# 设计文档：时间滑块精细化单位控制

## 概述

本设计实现了一个多级时间单位控制系统，将滑块轨道划分为多个速度区域，每个区域对应不同的时间单位（秒、分、时、天、月、年）。通过分段速度计算函数，实现从暂停到高速的平滑过渡，同时保持向后兼容性。

### 设计目标

1. 支持 7 个时间单位级别：暂停、秒/秒、分/秒、时/秒、天/秒、月/秒、年/秒
2. 确保速度在区域边界处连续且平滑过渡
3. 保持现有组件接口和交互行为不变
4. 优化性能，确保实时响应

### 核心设计决策

- **分段速度函数**：将滑块位置空间划分为 6 个区域（暂停区 + 5 个速度区），每个区域使用独立的速度计算公式
- **连续性保证**：在区域边界处，确保相邻区域的速度值相等
- **配置驱动**：所有区域边界、最大速度、曲线指数等参数均可通过配置文件调整

## 组件和接口

### 核心接口

```typescript
// 速度计算结果
interface SpeedResult {
  speed: number;           // 时间流速（单位/秒）
  direction: 'forward' | 'backward';  // 时间方向
}

// 速度区域配置
interface SpeedZoneConfig {
  start: number;           // 区域起始位置（归一化偏移量，0-1）
  end: number;             // 区域结束位置（归一化偏移量，0-1）
  maxSpeed: number;        // 区域最大速度
  unit: 'second' | 'minute' | 'hour' | 'day' | 'month' | 'year';  // 时间单位
  exponent: number;        // 速度曲线指数
}
```

### 核心函数

#### calculateSpeed(position: number): SpeedResult

根据滑块位置计算时间流速和方向。

**输入：**
- `position`: 滑块位置（0-1，0.5为中心点）

**输出：**
- `SpeedResult`: 包含速度和方向的对象

**算法：**
1. 将position转换为归一化偏移量：`offset = (position - 0.5) * 2`（范围：-1到1）
2. 检查是否在死区内：`|offset| < deadZone` → 返回速度0
3. 确定所在速度区域（根据|offset|的值）
4. 应用区域内的速度计算公式
5. 确定方向（offset > 0为前进，< 0为后退）

#### formatSpeedLabel(speed: number, lang: 'zh' | 'en'): string

格式化速度标签文本。

**输入：**
- `speed`: 速度值（单位/秒）
- `lang`: 语言（'zh'或'en'）

**输出：**
- 格式化的速度字符串

**逻辑：**
- 根据速度值自动选择合适的单位
- 根据语言选择对应的单位文本
- 对数值进行合理的精度控制

## 数据模型

### 速度区域划分

滑块轨道从中心（0.5）向两端划分为7个区域：

```
位置范围（归一化偏移量）    速度区域      时间单位    最大速度
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[-1.0, -0.8]  或  [0.8, 1.0]   区域6        年/秒      365天/秒
[-0.8, -0.6]  或  [0.6, 0.8]   区域5        月/秒      30天/秒
[-0.6, -0.4]  或  [0.4, 0.6]   区域4        天/秒      1天/秒
[-0.4, -0.2]  或  [0.2, 0.4]   区域3        时/秒      1/24天/秒
[-0.2, -0.1]  或  [0.1, 0.2]   区域2        分/秒      1/1440天/秒
[-0.1, -0.05] 或  [0.05, 0.1]  区域1        秒/秒      1/86400天/秒
[-0.05, 0.05]                  死区         暂停       0
```

**设计说明：**
- 中心为死区（暂停），宽度为归一化偏移量的0.1（即滑块位置的0.05）
- 从中心向外依次为：秒、分、时、天、月、年
- 每个区域的宽度相等（除死区外），便于用户操作
- 速度在区域内连续增长，在区域边界处连续

### 速度计算公式

对于每个速度区域，使用以下公式计算速度：

```
speed = minSpeed + (maxSpeed - minSpeed) * ((|offset| - zoneStart) / (zoneEnd - zoneStart))^exponent
```

其中：
- `offset`: 归一化偏移量（-1到1）
- `zoneStart`: 区域起始位置（归一化偏移量的绝对值）
- `zoneEnd`: 区域结束位置（归一化偏移量的绝对值）
- `minSpeed`: 区域最小速度（等于前一区域的最大速度，保证连续性）
- `maxSpeed`: 区域最大速度
- `exponent`: 速度曲线指数（控制加速度曲线）

**连续性保证：**
- 每个区域的`minSpeed`等于前一区域的`maxSpeed`
- 在区域边界处，速度值相等
- 使用指数曲线使速度变化更平滑

### 配置参数

所有配置参数存储在`visualConfig.ts`的`TIME_SLIDER_CONFIG`中：

```typescript
TIME_SLIDER_CONFIG = {
  // 死区配置
  deadZone: 0.05,  // 死区半径（归一化偏移量）
  
  // 速度区域配置
  speedZones: [
    {
      name: 'second',
      start: 0.05,
      end: 0.1,
      maxSpeed: 1 / 86400,  // 1秒/秒 = 1/86400天/秒
      exponent: 1.5,
      unit: { zh: '秒/秒', en: 's/s' }
    },
    {
      name: 'minute',
      start: 0.1,
      end: 0.2,
      maxSpeed: 1 / 1440,  // 1分/秒 = 1/1440天/秒
      exponent: 1.5,
      unit: { zh: '分/秒', en: 'min/s' }
    },
    {
      name: 'hour',
      start: 0.2,
      end: 0.4,
      maxSpeed: 1 / 24,  // 1时/秒 = 1/24天/秒
      exponent: 1.5,
      unit: { zh: '时/秒', en: 'h/s' }
    },
    {
      name: 'day',
      start: 0.4,
      end: 0.6,
      maxSpeed: 1,  // 1天/秒
      exponent: 2.0,
      unit: { zh: '天/秒', en: 'd/s' }
    },
    {
      name: 'month',
      start: 0.6,
      end: 0.8,
      maxSpeed: 30,  // 30天/秒
      exponent: 2.0,
      unit: { zh: '月/秒', en: 'm/s' }
    },
    {
      name: 'year',
      start: 0.8,
      end: 1.0,
      maxSpeed: 365,  // 365天/秒
      exponent: 2.5,
      unit: { zh: '年/秒', en: 'y/s' }
    }
  ]
}
```

**参数说明：**
- `deadZone`: 死区半径，控制暂停区域的大小
- `start/end`: 区域边界（归一化偏移量的绝对值）
- `maxSpeed`: 区域最大速度（统一使用天/秒作为内部单位）
- `exponent`: 速度曲线指数，控制加速度感觉
- `unit`: 显示单位的多语言文本

## 正确性属性

*属性是一个特征或行为，应该在系统的所有有效执行中保持为真——本质上是关于系统应该做什么的正式陈述。属性作为人类可读规范和机器可验证正确性保证之间的桥梁。*

### 属性 1：死区暂停

*对于任意*滑块位置，如果其归一化偏移量的绝对值小于死区半径，则计算出的速度应该为零。

**验证：需求 1.1, 1.2**

### 属性 2：速度连续性

*对于任意*两个相邻的速度区域，在它们的边界位置处，计算出的速度值应该相等（误差小于0.0001）。

**验证：需求 2.1, 2.3**

### 属性 3：速度单调性

*对于任意*方向（前进或后退），当滑块位置远离中心时，计算出的速度应该单调递增。

**验证：需求 2.4**

### 属性 4：速度范围约束

*对于任意*滑块位置，计算出的速度应该在0到最大速度（365天/秒）之间。

**验证：需求 1.5**

### 属性 5：方向一致性

*对于任意*滑块位置，如果位置大于0.5，则方向应该为前进；如果位置小于0.5，则方向应该为后退。

**验证：需求 6.1, 6.2**

### 属性 6：标签格式正确性

*对于任意*速度值和语言设置，格式化的标签应该包含数值和正确的单位文本。

**验证：需求 3.1-3.8**

### 属性 7：配置参数有效性

*对于任意*配置参数集合，所有区域边界应该连续且不重叠，所有速度值应该为非负数。

**验证：需求 4.1-4.5**

### 属性 8：双向对称性

*对于任意*滑块位置p，前进方向位置p和后退方向位置(1-p)应该产生相同的速度值（方向相反）。

**验证：需求 6.3**

## 错误处理

### 输入验证

1. **滑块位置验证**
   - 检查position是否在[0, 1]范围内
   - 超出范围时自动钳制到边界值
   - 记录警告日志

2. **配置参数验证**
   - 启动时验证所有区域边界的连续性
   - 验证速度值的非负性
   - 验证指数值的合理性（> 0）
   - 配置无效时使用默认配置并记录错误

### 运行时错误

1. **计算错误**
   - 捕获数学运算异常（如除零）
   - 返回安全的默认值（速度0）
   - 记录错误日志

2. **格式化错误**
   - 处理无效的语言参数（默认使用英文）
   - 处理NaN或Infinity速度值（显示"--"）
